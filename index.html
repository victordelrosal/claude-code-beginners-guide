Overview

What is Claude Code?

Claude Code is an AI assistant that lives in your terminal (command line). Unlike chatbots that give you text snippets to copy and paste, Claude Code works directly with your files, folders, and projects. Think of it as a capable colleague sitting at your computer who can read your code, understand your project, and make changes for you.

What makes it different from ChatGPT or regular Claude?

Regular AI Chat

You describe code, it gives you text. You copy, paste, and hope it works. It has no idea what your project looks like.

Claude Code

It reads your entire project. It edits your actual files. It runs your tests. It commits to Git. It sees the full picture.

The mental shift: You are not asking for code snippets. You are directing an AI teammate who has access to your project. Tell it what you want done, not how to do it.

Before You Begin

Who is this guide for?

This guide is for people who:

  • Have never used Claude Code before
  • May not be deeply technical (you do not need to be a programmer)
  • Are comfortable opening a terminal window
  • Want to use AI to be more productive with documents, code, data, or projects

What you will need

  • A Mac, Windows PC, or Linux machine
  • An internet connection
  • A Claude account (we will walk you through this next)
  • About 30 minutes for your first session

What is the terminal (and how do I open it)?

The terminal is a text-based window where you type commands instead of clicking buttons. It has been part of every computer since the 1970s. It looks intimidating at first, but you only need to know a handful of things to use Claude Code effectively.

When you open it, you will see a blinking cursor waiting for you to type. That is it. No menus, no icons. Just you and the computer having a text conversation.

How to open it on your computer

Mac: Press Cmd + Space to open Spotlight, type Terminal, and press Enter. You can also find it in Applications > Utilities > Terminal.

Windows: Press Win + X and select Terminal or PowerShell. Alternatively, search for "PowerShell" in the Start menu.

Linux: Press Ctrl + Alt + T on most distributions. Or find "Terminal" in your applications menu.

The only commands you need to know

Before using Claude Code, there are just two terminal commands worth knowing:

cd folder-name Move into a folder (cd = "change directory")
ls List the files in your current folder (on Windows, use dir)

That is genuinely all you need. Once you type claude, the AI takes over and you communicate in plain English from that point on.

Do not worry if the terminal feels unfamiliar. Claude Code is designed to work through natural language. You will not need to memorise commands or learn shell scripting. If you can type a sentence, you can use Claude Code.

2

Install Claude Code

Open your terminal and run the installer for your platform.

macOS or Linux

$ curl -fsSL https://claude.ai/install.sh | bash

If you prefer Homebrew on Mac:

$ brew install claude-code

Windows (PowerShell)

PS> irm https://claude.ai/install.ps1 | iex

Windows requires Git for Windows to be installed first.

Verify it worked

$ claude --version claude-code v1.x.x

Note: The installer is lightweight and handles updates automatically. You do not need Node.js, Python, or any other runtime pre-installed.

3

Your first session

Navigate to any project folder and type claude.
$ cd ~/my-project $ claude

On first launch, Claude will ask you to log in through your browser. After authentication, you are in an interactive session. You will see a prompt where you can type in natural language, just as you would to a colleague.

Try asking it something

claude> What files are in this project? Give me a high-level overview.

Claude will read your project files and give you a structured summary. It understands file types, project structure, and can explain what each part does.

Try something more specific

claude> Find all the places where we handle user authentication

Claude searches through your codebase and shows you exactly where things happen. No grep commands, no scrolling through files. Just ask.

4

The permission system

Claude Code always asks before changing anything. You stay in control.

When Claude wants to edit a file, create a file, or run a command, it will show you exactly what it plans to do and ask for permission. This is one of its best safety features.

Claude wants to edit: src/config.js - const timeout = 5000; + const timeout = 10000; Allow? [y/n/always]

Your options:

  • y (yes): Allow this one change
  • n (no): Reject it, Claude will try a different approach
  • always: Allow this type of action for the rest of the session

Golden rule: Read the diffs before pressing yes. Especially in your first few sessions, take a moment to understand what Claude is proposing. This is how you learn its strengths and build trust.

5

Your first real task

Four beginner-friendly tasks you can try right now.

Task A: Understand unfamiliar code

claude> Explain what the function processPayment in src/billing.js does, step by step, in plain language.

Perfect for when you inherit a codebase or join a new project.

Task B: Fix a bug

claude> Users are reporting that the login form accepts empty passwords. Find where password validation happens and fix it.

Claude will search, identify the issue, propose a fix, and apply it (with your approval).

Task C: Add a feature

claude> Add a "last modified" timestamp that shows at the bottom of each blog post page.

Claude will find the relevant template, add the timestamp, and explain what it did.

Task D: Work with data

claude> Read the CSV file in data/students.csv and create a summary showing the average grade per course.

Claude can read data files, write analysis scripts, and generate results.

6

Essential commands

You do not need to memorize these. But they are useful to know.

Starting Claude Code

claude Start interactive session in current folder
claude "do X" Run a single task, then exit
claude -c Continue your most recent session

Inside a session

/help See available commands
/clear Clear the current conversation
/compact Summarize the conversation to save context
Esc Cancel a running operation
Ctrl+C Exit Claude Code

Working with Git

claude commit Stage and commit changes with an AI-written message

Pro tip: You do not need slash commands for most things. Just type what you want in natural language. "Commit my changes" or "Show me what changed" work perfectly well.

8

CLAUDE.md: teaching Claude about your project

Your project's instruction manual for Claude. Optional but powerful.

Create a file called CLAUDE.md in the root of your project. Claude reads it at the start of every session. It is where you tell Claude things like your coding style, project conventions, or things to avoid.

Example CLAUDE.md

# Project: Student Portal ## Tech Stack This is a Python Flask app with PostgreSQL. We use pytest for testing. ## Conventions - Use snake_case for all Python functions - Every new endpoint needs a test - Never modify the migration files directly ## Important - The database credentials are in .env (never commit this file) - Run tests with: pytest tests/ -v

Think of it as onboarding documentation, not for a new hire, but for your AI teammate. The more context you give, the better Claude performs.

You can also place a CLAUDE.md in your home directory (~/.claude/CLAUDE.md) for preferences that apply to all your projects, such as your preferred language, formatting style, or general rules.

Tips from the trenches

Practical advice from daily Claude Code use.

Be specific about what you want

Instead of "fix the login", try "the login form on /login should show an error message when the email field is empty, instead of silently submitting". The more precise your request, the better the result.

Start small

Do not ask Claude to rewrite your entire application in the first session. Start with small, contained tasks: fix a bug, add a feature, refactor one function. Build confidence and understanding gradually.

Review the diffs

Always read what Claude proposes before accepting. This is not just about safety: it is the fastest way to learn. You will start recognising patterns, understanding your codebase better, and developing intuition for what good AI-assisted work looks like.

Use continue mode

If you close your terminal, claude -c picks up where you left off. Your conversation context is preserved. No need to re-explain what you were working on.

Work in a project folder

Always cd into your project directory before running claude. Claude uses your current folder as its workspace. If you start Claude from your home directory, it has access to everything (which is rarely what you want).

Let Claude run commands

Claude can run shell commands (with your permission). Let it run tests, install packages, check git status, or build your project. This is where the real power is: it can see errors, fix them, and verify the fix all in one flow.

10

What to try next

Once you are comfortable with the basics, explore these areas.
  • Git workflows: Ask Claude to review your changes, write commit messages, or create pull requests
  • Testing: "Write tests for the user registration module" is a perfectly valid request
  • Refactoring: "This function is too long. Break it into smaller, well-named functions"
  • Documentation: "Generate API documentation for the endpoints in routes/"
  • Code review: "Review this file for potential bugs or security issues"
  • Data analysis: "Read this Excel file and create a chart showing monthly trends"
  • Learning: "Explain how async/await works in this codebase with real examples from our code"
